TITLE: Signal Correlator and Anomaly Detector
DESCRIPTION: A stateful subsystem that correlates cross-agent signal streams using a sliding-window algorithm to surface anomalous patterns, causal clusters, and entropy spikes in real time. Designed as a passive observer that enriches the Ichor signals nervous system with higher-order intelligence.
SUBSYSTEM: Ichor.Subsystems.SignalCorrelator
SIGNAL_INTERFACE: Subscribes to all Signals categories via Signals.subscribe/1; emits correlated anomaly and cluster signals back through Signals.emit/2
TOPIC: subsystem:signal_correlator
VERSION: 0.1.0
FEATURES: sliding-window signal buffering, cross-agent correlation matrix, entropy spike detection, causal cluster grouping, anomaly severity scoring, rate-of-change tracking, PubSub anomaly broadcast
USE_CASES: detect agent storm patterns before cascade failure, surface correlated errors across fleet members, alert on entropy spikes during high-load events, identify causal chains in multi-agent workflows, feed anomaly data into DashboardLive for real-time operator visibility
ARCHITECTURE: GenServer holding a ring buffer (ETS) of the last N signals per category; a correlation worker that runs on configurable tick interval (default 5s) computing pairwise Pearson correlation across agent event streams; an anomaly classifier that thresholds on z-score deviations; results published as structured Signals.Message envelopes with severity metadata; supervised under Ichor.Subsystems.Supervisor
DEPENDENCIES: Ichor.Signals, Ichor.Signals.Bus, Ichor.Signals.Catalog, Ichor.Signals.Message, Ichor.Signals.Buffer, Phoenix.PubSub
SIGNALS_EMITTED: :anomaly_detected, :correlation_cluster_formed, :entropy_spike, :signal_storm_warning, :correlator_tick
SIGNALS_SUBSCRIBED: :agent_started, :agent_stopped, :agent_error, :message_sent, :message_received, :task_started, :task_completed, :heartbeat_tick, :fleet_sweep_complete
