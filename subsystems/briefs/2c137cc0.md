TITLE: Signal Correlation and Anomaly Detection Engine
DESCRIPTION: A dual-mode subsystem that correlates signals emitted across the agent fleet to detect causality chains and coordinated state changes, while continuously monitoring signal streams for statistical anomalies using adaptive baselines. Bridges signal correlation (temporal reasoning) and self-healing (anomaly-triggered recovery) into a unified intelligence layer.
SUBSYSTEM: Ichor.Subsystems.SignalCorrelator
SIGNAL_INTERFACE: Subscribes to all signal categories via wildcard; emits correlation and anomaly events; triggers self-healing actions via fleet signals
TOPIC: subsystem:signal_correlator
VERSION: 0.1.0
FEATURES: sliding-window cross-correlation, CUSUM anomaly detection, adaptive baseline learning, causality chain reconstruction, fleet-wide pattern registry, self-healing trigger emission
USE_CASES: detect cascading agent failures before they propagate, identify correlated signal bursts indicating coordinated state changes, trigger automatic recovery when anomaly thresholds are crossed, surface temporal patterns for Archon analysis, build causality maps for post-incident forensics
ARCHITECTURE: GenServer holding a circular buffer of recent signals per agent; background Task for correlation passes every 500ms; ETS table for baseline statistics per signal atom; Phoenix.PubSub subscriber for all signal categories; emits structured events when correlation score exceeds threshold or CUSUM statistic exceeds control limit
DEPENDENCIES: Ichor.Registry, Ichor.Signals.Catalog, Phoenix.PubSub, Ichor.Fleet.AgentProcess
SIGNALS_EMITTED: correlation_pattern_detected, causality_chain_identified, anomaly_detected, anomaly_cleared, baseline_updated, self_heal_triggered
SIGNALS_SUBSCRIBED: agent_spawned, agent_terminated, agent_heartbeat, run_started, run_completed, entropy_alert, fleet_state_changed
