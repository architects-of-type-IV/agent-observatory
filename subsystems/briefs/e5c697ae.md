TITLE: Signal Correlation Engine
DESCRIPTION: Cross-agent signal correlation subsystem that detects temporal patterns across the fleet, identifies cascading failures before they propagate, and emits remediation signals to trigger self-healing responses. Combines sliding-window correlation with anomaly scoring to give the Ichor control plane early-warning and auto-recovery capabilities.
SUBSYSTEM: Ichor.Subsystems.SignalCorrelator
SIGNAL_INTERFACE: Subscribes to agent:signals:* wildcard topic; correlates :heartbeat, :error, :degraded, :overload, :recovered signals across agent population
TOPIC: subsystem:signal_correlator
VERSION: 0.1.0
FEATURES: sliding-window temporal correlation, multi-agent pattern matching, anomaly score computation, self-healing trigger emission, configurable correlation window (default 30s), per-team and fleet-wide correlation scopes
USE_CASES: detect cascading failure before full fleet impact, auto-restart degraded agents when correlation score exceeds threshold, surface correlated anomalies in dashboard fleet view, feed entropy scores into Archon planning context
ARCHITECTURE: GenServer holding a ring-buffer of recent signals per agent (ETS-backed for read performance). On each new signal, recomputes Pearson correlation coefficients across active agent signal streams within the current time window. When correlation score for :error or :degraded signals exceeds the configured threshold, emits a :correlation_alert signal with affected agent list and recommended remediation action. Subscribes to PubSub on startup; broadcasts alerts back onto subsystem:signal_correlator topic.
DEPENDENCIES: Ichor.Registry, Ichor.Gateway.Operator, Phoenix.PubSub, Ichor.Fleet
SIGNALS_EMITTED: :correlation_alert, :self_heal_trigger, :fleet_degraded, :correlation_cleared
SIGNALS_SUBSCRIBED: :heartbeat, :error, :degraded, :overload, :recovered, :agent_joined, :agent_left
