TITLE: MES Signal Anomaly and Correlation Engine
DESCRIPTION: Real-time anomaly detection and cross-signal causal correlation for MES sensor streams, combining EWMA control charts with context-aware suppression during recipe transitions and Pearson cross-correlation for fleet-wide causal chain detection.
SUBSYSTEM: Ichor.Mes.SignalAnomalyEngine
SIGNAL_INTERFACE: :mes_anomaly_config
TOPIC: subsystem:mes_signal_anomaly_engine
VERSION: 0.1.0
FEATURES: EWMA baseline tracking per station/metric, 2-sigma warning / 3-sigma critical / 5-sigma halt severity tiers, 30s transition mute window for scheduled recipe changes, sliding-window Pearson cross-correlation across agent signal pairs, causal lag detection (-5s to +5s at 500ms steps), ETS-backed O(1) per-station-metric state, anomaly cleared signal on recovery
USE_CASES: detect sensor drift before production halt, correlate upstream station failures to downstream effects, suppress spurious alerts during scheduled recipe transitions, identify causal chains between agent metric degradation events
ARCHITECTURE: Single GenServer with two ETS tables (anomaly state per {station_id, metric} key; correlation ring buffers per agent pair). Two detection paths fan out from a shared PubSub subscriber: (1) per-signal EWMA control chart with alpha=0.2, rolling sigma, and transition mute gate; (2) pairwise cross-correlation with 60s sliding window computing Pearson r at lag offsets, flagging |r|>0.85 at non-zero lag as causal. Implements Ichor.Mes.Subsystem behaviour (info/0, start/0, handle_signal/1, stop/0).
DEPENDENCIES: Ichor.Signals, Phoenix.PubSub, Ichor.Fleet.AgentProcess
SIGNALS_EMITTED: :mes_anomaly_detected, :mes_anomaly_cleared, :mes_correlation_alert
SIGNALS_SUBSCRIBED: :mes_sensor_reading, :mes_schedule_transition, :agent_metric, :mes_anomaly_config
