TITLE: Adaptive Load Balancer
DESCRIPTION: Dynamically routes new task assignments to the least-loaded capable agent by monitoring real-time fleet workload signals, preventing hotspots and enabling automated fleet optimization.
SUBSYSTEM: Ichor.Balancer
SIGNAL_INTERFACE: Controlled via Ichor.Signals.Catalog; subscribes to fleet and activity signals; emits load_balanced and agent_overloaded signals
TOPIC: subsystem:adaptive_load_balancer
VERSION: 0.1.0
FEATURES: Real-time ETS load tracking per agent, capability-aware task routing, PubSub-driven score updates, configurable balancing strategies, overload circuit breaker, fleet capacity signal emission
USE_CASES: Route new tasks to least-loaded capable agent, detect fleet hotspots, rebalance tasks during agent saturation, emit alerts on capacity breach, graceful degradation under high load
ARCHITECTURE: GenServer holding ETS table of per-agent load scores (task count, last-event-at, capability tags); subscribes to fleet signals via Ichor.Signals.Bus; exposes route/1 public API; emits signals on balance decisions and overload events; supervised under Ichor.Fleet.FleetSupervisor
DEPENDENCIES: Ichor.Fleet, Ichor.Signals, Ichor.Signals.Bus, Ichor.Signals.Catalog, Ichor.Fleet.FleetSupervisor
SIGNALS_EMITTED: :load_balanced, :agent_overloaded, :fleet_capacity_critical
SIGNALS_SUBSCRIBED: :agent_event_received, :agent_heartbeat, :task_assigned, :task_completed
