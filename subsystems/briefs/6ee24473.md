TITLE: Signal Journal — File-Based Signal Persistence
DESCRIPTION: A supervised GenServer that buffers PubSub signals in memory and periodically flushes them to rotating JSONL log files on disk, then writes a daily markdown digest summarizing fleet activity by category — the only file-based signal persistence path in the system.
SUBSYSTEM: Ichor.Subsystems.SignalJournal
SIGNAL_INTERFACE: Subscribes to configurable topic list at init via Phoenix.PubSub; controlled via :journal_start, :journal_stop, :journal_flush_now, :journal_set_topics signals
TOPIC: subsystem:signal_journal
VERSION: 0.1.0
FEATURES: configurable topic subscription, in-memory ring buffer, periodic JSONL flush every N minutes (default 5), daily digest rollup to markdown, rotating files per calendar day, file handles opened/closed per flush (no stale handles), emits journal:flushed and journal:digest_written signals
USE_CASES: operator reviews signals-2026-03-20.jsonl to audit yesterday's fleet activity, dashboard subscribes to journal:flushed to display last flush timestamp and entry count, WebhookEgress subscribes to journal:flushed to ship completed logs offsite, post-incident forensics reads JSONL to reconstruct agent event sequence
ARCHITECTURE: Single GenServer holding state: %{buffer: [], topics: [], flush_interval_ms: integer, log_dir: Path.t(), last_flush: DateTime.t()}. On init: subscribes to all configured topics, schedules first :flush via Process.send_after. handle_info for PubSub messages: prepends timestamped JSON entry to buffer. handle_info(:flush): opens signals-YYYY-MM-DD.jsonl in append mode, writes buffer as JSONL, closes handle, broadcasts journal:flushed, reschedules next flush. handle_info(:daily_digest): reads prior day file, groups by signal category, counts per category, writes digest-YYYY-MM-DD.md with category count table and top-5 events, broadcasts journal:digest_written. Implements Ichor.Mes.Subsystem: info/0 returns metadata map, start/0 calls GenServer.start_link, handle_signal/1 routes :journal_flush_now/:journal_set_topics/:journal_stop control signals, stop/0 triggers final flush then shutdown.
DEPENDENCIES: Phoenix.PubSub, Jason, Ichor.Signals.Catalog, Ichor.SystemSupervisor
SIGNALS_EMITTED: journal:flushed, journal:digest_written
SIGNALS_SUBSCRIBED: fleet:agent_started, fleet:agent_stopped, fleet:agent_crashed, team:launched, team:completed, mes:run_completed, mes:run_failed, hitl:gate_triggered, monitoring:alert, agent:tool_called, journal:flush_now, journal:set_topics, journal:stop
