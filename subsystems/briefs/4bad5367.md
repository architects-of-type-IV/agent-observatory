TITLE: Ichor Signal Scheduler
DESCRIPTION: A cron-like GenServer subsystem that emits configured PubSub signals on repeating intervals, enabling time-driven fleet actions (heartbeats, sweeps, run lifecycle events) without any external process manager or SaaS dependency.
SUBSYSTEM: Ichor.Subsystems.SignalScheduler
SIGNAL_INTERFACE: Subscribes to mes:scheduler_config for runtime schedule management; emits arbitrary signals on caller-configured topics
TOPIC: subsystem:signal_scheduler
VERSION: 0.1.0
FEATURES: interval-based signal emission, runtime add/remove of schedule entries without restart, ETS-backed schedule registry, crash-safe rescheduling via handle_info :tick, O(1) entry lookup by ID
USE_CASES: fleet heartbeat sweeps on fixed intervals, periodic mes:run_started probes, monitoring:sweep triggers, scheduled agent health checks
ARCHITECTURE: Single GenServer owns one named ETS table keyed by schedule ID. start/0 seeds entries from application config and sends self a :tick_{id} message per entry via Process.send_after. Each handle_info(:tick_{id}) broadcasts {topic, signal} via Phoenix.PubSub then reschedules itself. Runtime adds insert into ETS and start the tick loop; removes delete from ETS and the next :tick is a no-op (entry missing = skip + no reschedule). Implements Ichor.Mes.Subsystem behaviour: info/0, start/0, handle_signal/1, stop/0.
DEPENDENCIES: Phoenix.PubSub, Ichor.Mes.Subsystem
SIGNALS_EMITTED: scheduler.entry_added, scheduler.entry_removed, scheduler.ticked, scheduler.stopped
SIGNALS_SUBSCRIBED: mes:scheduler_config
