TITLE: Ichor Signal Intelligence - Anomaly Detection and Causal Correlation
DESCRIPTION: A two-tier subsystem that detects statistical anomalies in per-agent telemetry streams using CUSUM-EWMA analysis, then discovers causal relationships between anomaly signals using sliding-window cross-correlation, enabling Ichor to surface hidden failure dependencies and emergent cascade patterns without external ML services.
SUBSYSTEM: Ichor.Subsystems.SignalIntelligence
SIGNAL_INTERFACE: subscribes to :agent_metric_reported and all PubSub signal categories (wildcard); emits :agent_anomaly_detected, :agent_anomaly_cleared, :fleet_anomaly_summary, :signal_correlated, :causal_chain_detected; accepts config signals :correlator_set_window and :correlator_set_threshold
TOPIC: subsystem:signal_intelligence
VERSION: 0.1.0
FEATURES: per-agent CUSUM-EWMA anomaly scoring, adaptive baseline recalibration to avoid alert fatigue, fleet-wide Isolation Forest batch analysis via Broadway pipeline, sliding-window cross-correlation with autocorrelation whitening, causal chain DAG detection for 3+ linked signals, ETS-backed ring buffers for zero-copy signal history, PubSub signal emission with severity tiers
USE_CASES: detect agent stalls before they become hard failures, discover that :memory_pressure predicts :agent_stalled by N seconds, alert operators to correlated failure cascades across teams, provide signal data to self-healing responders, surface emergent causal patterns without manual rule authoring
ARCHITECTURE: Two supervised GenServer tiers. AnomalyServer: one child GenServer per AgentProcess holding EWMA/CUSUM accumulators (3 floats per metric: ewma, cusum_pos, cusum_neg); a Broadway pipeline aggregates fleet metrics on 60s rolling windows for batch Isolation Forest outlier scoring. CorrelatorServer: ETS ring buffer per signal type with timestamps, :tick every 5s triggers batch cross-correlation with full lag sweep (-window to +window), whitens autocorrelation via EMA before scoring, lightweight DAG builder emits :causal_chain_detected when A->B and B->C correlations exist. Both tiers supervised under a dedicated SignalIntelligenceSupervisor with :one_for_one strategy.
DEPENDENCIES: Ichor.AgentProcess, Ichor.Fleet, Ichor.Registry, Phoenix.PubSub, Broadway
SIGNALS_EMITTED: :agent_anomaly_detected, :agent_anomaly_cleared, :fleet_anomaly_summary, :signal_correlated, :causal_chain_detected
SIGNALS_SUBSCRIBED: :agent_metric_reported, :correlator_set_window, :correlator_set_threshold, wildcard_all_signal_categories
