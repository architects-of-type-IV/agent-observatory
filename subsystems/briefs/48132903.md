TITLE: ContextTide — Shared Context Distillation Engine
DESCRIPTION: ContextTide maintains a time-windowed pool of shared observations, decisions, and signals from all agents in a team. As shared context grows, it automatically distills and rebroadcasts compressed summaries to all team members, preventing context fragmentation in long-running multi-agent sessions.
SUBSYSTEM: Ichor.Subsystems.ContextTide
SIGNAL_INTERFACE: :context_fragment_received, :context_distill_requested, :context_tide_broadcast
TOPIC: subsystem:context_tide
VERSION: 0.1.0
FEATURES: sliding time-window context pool, LLM-powered distillation, team-scoped broadcast, configurable tide threshold, per-agent context delta delivery, distillation audit log
USE_CASES: prevent agent context divergence in long team sessions, summarize shared research findings across researcher agents, maintain coherent shared memory during multi-phase DAG pipelines, surface common reasoning threads before planner synthesis
ARCHITECTURE: GenServer per active team holds a ring buffer of context fragments (team_id -> [{agent_id, timestamp, fragment}]). A tide_level counter tracks total tokens. When tide_level exceeds threshold, a Task calls the LLM distillation step, replaces the buffer with the compressed summary, and broadcasts :context_tide_broadcast on subsystem:context_tide. Agents subscribe on join and receive delta updates via PubSub.
DEPENDENCIES: Ichor.Fleet, Ichor.Signals.Catalog, Phoenix.PubSub, Ichor.Archon.Chat
SIGNALS_EMITTED: :context_tide_broadcast, :context_distilled, :context_tide_overflow
SIGNALS_SUBSCRIBED: :agent_message_sent, :context_fragment_received, :context_distill_requested, :team_agent_joined
