---

## DESCRIPTION

`Ichor.Nervous.Correlator` is a real-time signal correlation engine and self-healing
orchestrator for ICHOR IV. It consumes signals from all fleet categories, runs sliding
temporal windows to detect causal multi-signal patterns (e.g. `agent_error` + `high_entropy`
+ `latency_spike` within 30s), and automatically emits remediation signals that trigger
corrective actions — restart, rebalance, quarantine, or Architect escalation — before
failures cascade across the fleet.

## SIGNAL_INTERFACE

Subscribes to `:all` signal categories via `Ichor.Signals.subscribe/1`.
Emits signals under the `:correlator` category:
- `correlator.pattern_matched` — a registered causal pattern fired
- `correlator.healing_applied` — a remediation action was executed
- `correlator.healing_escalated` — severity exceeded auto-heal threshold; Architect notified
- `correlator.pattern_registered` — a new detection pattern was registered at runtime
- `correlator.circuit_open` — a session's circuit breaker opened (too many failures)
- `correlator.circuit_closed` — circuit breaker reset after recovery window

Control signals (received via `Ichor.Signals.emit/3` with scope):
- `correlator:register_pattern` — add a new causal chain pattern at runtime
- `correlator:set_remediation` — attach a remediation action to a pattern
- `correlator:override_circuit` — manually open/close a circuit breaker

## FEATURES

- Sliding temporal window correlation (configurable 10s–300s windows)
- NFA-based multi-signal pattern matching (ordered and unordered variants)
- Per-session circuit breakers (closed → open → half-open state machine)
- Graduated remediation ladder: warn → nudge → restart → quarantine → escalate
- Pattern registry: register/deregister causal patterns at runtime via signals
- ETS ring buffer per session (last 500 signals, O(1) window scans)
- Zero-allocation hot path: pattern matching on pre-compiled NFA states
- Heartbeat-driven circuit-breaker sweeps (resets half-open circuits after TTL)
- Dashboard integration: `:correlator` view mode with active patterns + circuit status

---

## USE_CASES

- **Cascade prevention**: detect `fleet.agent.error` + `fleet.entropy.spike` + `fleet.agent.error` from 3 different sessions within 60s → emit `correlator.pattern_matched` → auto-restart the highest-entropy session
- **Stuck detection**: detect `agent.heartbeat` absent for 5 consecutive beats → open circuit → quarantine session → escalate to Architect if quarantine persists >2min
- **Resource exhaustion**: correlate `cost.budget_exceeded` + `fleet.agent.spawned` within 30s → circuit-break spawning for that team
- **Runaway agent**: detect `fleet.agent.spawned` > N times in 60s from same session → open spawn circuit breaker → emit escalation signal
- **Recovery confirmation**: when `fleet.agent.healthy` follows a circuit-open event → transition to half-open → confirm and close after 3 healthy beats

## ARCHITECTURE

```
Ichor.Nervous.Correlator (GenServer — main loop, owns ETS tables)
  ├── PatternRegistry   (ETS: pattern_id → %NfaPattern{})
  ├── WindowBuffer      (ETS: session_id → :queue of {signal, ts})
  ├── CircuitTable      (ETS: session_id → %Circuit{state, failure_count, opened_at})
  └── RemediationEngine (GenServer — executes actions, rate-limits escalations)

Ichor.Nervous.Correlator.Supervisor (one_for_one, permanent)
Ichor.Nervous.Correlator.PatternCompiler (pure module — compiles pattern DSL → NFA states)
```

Signal flow:
1. All signals arrive at `handle_info/2` via `Signals.subscribe(:all)`
2. Signal appended to per-session WindowBuffer (ETS queue, capped at 500)
3. PatternRegistry checked: for each active pattern, run NFA against window
4. Match → RemediationEngine executes configured action → emit remediation signal
5. CircuitTable updated on every error/recovery signal
6. Heartbeat tick: sweep CircuitTable for half-open expiry, emit `correlator.circuit_closed`

NFA pattern example (Elixir DSL):
```elixir
Correlator.register_pattern(:cascade_risk,
  sequence: [:agent_error, :entropy_spike, :agent_error],
  window_ms: 60_000,
  scope: :fleet,
  remediation: {:restart_highest_entropy, []},
  severity: :high
)
```
