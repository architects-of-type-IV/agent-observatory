TITLE: Ichor.Subsystems.SseSignalGateway
DESCRIPTION: Exposes Ichor's internal PubSub bus as an HTTP Server-Sent Events stream, allowing any external consumer (curl, browser EventSource, n8n, Grafana Infinity) to subscribe to a live signal feed via a plain HTTP GET with optional per-request topic filtering — no pre-configuration required.
PLUGIN: Ichor.Subsystems.SseSignalGateway
SIGNAL_INTERFACE: Subscribes to operator-configured PubSub topics filterable per-request via ?topics= query param; emits client lifecycle signals on join/leave/count-change
TOPIC: plugin:sse_signal_gateway
VERSION: 0.1.0
FEATURES: per-request topic filtering via query param, max-connections cap enforced by GenServer, ETS client registry, clean disconnect handling, plain HTTP with no special client required, zero external dependencies
USE_CASES: curl -N live signal inspection, browser EventSource dashboards, n8n HTTP trigger integration, Grafana Infinity datasource bridging, ad-hoc operator debugging of live signal bus
ARCHITECTURE: Two modules — (1) a Phoenix controller action that sets Content-Type text/event-stream, calls send_chunked(200), subscribes to requested PubSub topics, and enters a receive loop forwarding each signal as an SSE data frame; on chunk error it unsubscribes and exits cleanly. (2) SseSignalGateway GenServer that owns an ETS client registry, enforces configurable max-connections cap, and emits client lifecycle signals on connect/disconnect events.
DEPENDENCIES: Ichor.SignalBus, Ichor.Signals.Catalog, Phoenix.Controller, Phoenix.PubSub
SIGNALS_EMITTED: signals.sse.client_connected, signals.sse.client_disconnected, signals.sse.client_count_changed
SIGNALS_SUBSCRIBED: operator-configured topic patterns (e.g. fleet:*, agent:*, mes:*, signals:*); default subscribes to all categories if no topics param provided
