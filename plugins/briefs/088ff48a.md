TITLE: Ichor.Plugins.WebhookNotifier
DESCRIPTION: Outbound HTTP push plugin that fires a JSON POST to one or more user-configured URLs whenever a subscribed PubSub signal fires, enabling external systems to react to internal Ichor events without polling.
PLUGIN: Ichor.Plugins.WebhookNotifier
SIGNAL_INTERFACE: Subscribes to a configurable list of PubSub topics (e.g. mes:run_complete, fleet:agent_crashed, hitl:paused); emits webhook:delivered and webhook:failed back onto the bus after each dispatch attempt.
TOPIC: plugin:webhook_notifier
VERSION: 0.1.0
FEATURES: configurable topic subscription list, configurable URL list, JSON serialization via Jason, supervised HTTP dispatch via Req, exponential backoff retry (1s/2s/4s, max 3 attempts), webhook:delivered signal on success, webhook:failed signal on non-2xx or connection error, fully stateless dispatch (no database writes)
USE_CASES: notify n8n/Zapier/custom HTTP listeners when a MES run completes, alert an external monitoring endpoint when an agent crashes, trigger downstream automation on hitl:paused, chain webhook:failed into a future AlertEscalator plugin
ARCHITECTURE: Single GenServer implementing Ichor.Plugin behaviour. start/0 reads application config (:topics, :urls, :max_retries), subscribes to each topic via Phoenix.PubSub.subscribe/2, and starts a supervised Req/Finch HTTP pool. handle_signal/1 receives each matching signal, serializes it to JSON with Jason.encode!/1, and POSTs to every configured URL with Content-Type application/json and Req retry step for backoff. After each URL attempt, emits webhook:delivered (with url, status_code, original_topic) or webhook:failed (with url, reason, original_topic) onto the bus. No mutable state beyond the Req pool. Under 150 lines of Elixir.
DEPENDENCIES: Phoenix.PubSub, Jason, Req
SIGNALS_EMITTED: webhook:delivered, webhook:failed
SIGNALS_SUBSCRIBED: configurable at runtime via application config (defaults: mes:run_complete, fleet:agent_crashed, hitl:paused)
