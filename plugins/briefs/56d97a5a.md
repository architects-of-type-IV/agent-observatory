TITLE: Desktop Notifier
DESCRIPTION: Fires a native OS desktop notification whenever a subscribed PubSub signal fires, giving the operator an immediate local alert when a MES run finishes, an agent crashes, or any other configured signal of interest fires — without polling the dashboard. Zero external dependencies; uses osascript on macOS and notify-send on Linux.
PLUGIN: Ichor.Plugins.DesktopNotifier
SIGNAL_INTERFACE: Subscribes to a user-configurable list of PubSub topics (default: mes:run_complete, fleet:agent_crashed, hitl:intervention_required). Emits no signals — notification delivery to the OS is the sole output.
TOPIC: plugin:desktop_notifier
VERSION: 0.1.0
FEATURES: native OS desktop notification dispatch, macOS (osascript) + Linux (notify-send) platform support, async dispatch via Task.Supervisor to avoid blocking the subscriber loop, configurable signal subscription list, configurable title/body field extraction from signal payload, graceful no-op on headless servers (detects DISPLAY/TERM_PROGRAM absence at start and logs a warning instead of crashing), per-notification debounce window to prevent OS notification spam during signal bursts
USE_CASES: operator receives a desktop popup the moment a long MES run completes even with the browser tab buried, operator is alerted immediately when an agent crashes without watching the dashboard, HITL intervention requests surface as OS-level notifications requiring no active monitoring, any downstream plugin can emit a signal that DesktopNotifier will surface to the human operator
ARCHITECTURE: Single GenServer (Ichor.Plugins.DesktopNotifier) started under Ichor.Plugins.Supervisor. On start/0 it detects the host platform (mac/linux/headless), subscribes to all configured PubSub topics via Phoenix.PubSub.subscribe/2, and stores platform + config in GenServer state. handle_info/2 receives each signal, extracts title and body fields from the payload map using configured keys, then delegates dispatch to a Task.Supervisor child (Ichor.Plugins.DesktopNotifier.TaskSupervisor) so slow osascript/notify-send calls never block the subscriber loop. Module layout: lib/ichor/plugins/desktop_notifier.ex (GenServer + Plugin behaviour impl), lib/ichor/plugins/desktop_notifier/dispatcher.ex (platform dispatch logic + System.cmd calls). Config read via Ichor.Settings or plugin manifest map passed to start/1.
DEPENDENCIES: Phoenix.PubSub, Ichor.Signals (for signal shape), Ichor.Plugins (behaviour), Task.Supervisor (already in OTP app supervisor tree)
SIGNALS_EMITTED: none
SIGNALS_SUBSCRIBED: mes:run_complete, fleet:agent_crashed, hitl:intervention_required (user-configurable; defaults shown)

MODULE STRUCTURE:
  lib/ichor/plugins/desktop_notifier.ex
    - defmodule Ichor.Plugins.DesktopNotifier
    - @behaviour Ichor.Plugin
    - info/0 :: %{name:, version:, description:}
    - start/0 :: {:ok, pid} | {:error, term}
    - stop/0 :: :ok
    - handle_signal/1 :: :ok  (delegates to handle_info via PubSub)
    - init/1 :: detects platform, subscribes to topics, stores state
    - handle_info({:signal, payload}, state) :: async dispatch
  lib/ichor/plugins/desktop_notifier/dispatcher.ex
    - dispatch(:mac, title, body) :: System.cmd("osascript", [...])
    - dispatch(:linux, title, body) :: System.cmd("notify-send", [...])
    - dispatch(:headless, _title, _body) :: :noop

FILE PATHS:
  lib/ichor/plugins/desktop_notifier.ex
  lib/ichor/plugins/desktop_notifier/dispatcher.ex

SUPERVISION WIRING:
  Ichor.Plugins.Supervisor children:
    {Task.Supervisor, name: Ichor.Plugins.DesktopNotifier.TaskSupervisor}
    {Ichor.Plugins.DesktopNotifier, config}
  DesktopNotifier started after its TaskSupervisor so dispatch is always available.

CONFIG SCHEMA:
  %{
    signals: ["mes:run_complete", "fleet:agent_crashed"],
    title_field: :run_id,
    body_field: :summary,
    debounce_ms: 2000
  }

ACCEPTANCE CRITERIA:
  1. mix compile --warnings-as-errors passes
  2. On macOS: starting the plugin and sending a mes:run_complete PubSub message triggers an osascript notification visible on the desktop
  3. On headless (no TERM_PROGRAM and no DISPLAY env vars): plugin starts successfully, logs a warning, and does not crash when a signal fires
  4. Dispatch is async: a 500ms sleep in the dispatcher does not delay handle_info return
  5. Debounce: two signals on the same topic within debounce_ms window result in only one OS notification
  6. info/0 returns %{name: "DesktopNotifier", version: "0.1.0", description: "..."}
  7. stop/0 unsubscribes from all PubSub topics cleanly

EDGE CASES:
  - osascript not found on PATH: catch {:error, :enoent} from System.cmd, log warning, do not crash
  - Signal payload missing the configured title_field or body_field: fall back to inspect(payload) as body, topic string as title
  - Rapid burst of signals (e.g. 50 agent_crashed in 200ms): debounce_ms window collapses to one notification per topic per window; TaskSupervisor ensures no unbounded process accumulation
  - Plugin restarted mid-flight: start/0 re-subscribes idempotently (PubSub subscribe is safe to call multiple times from the same process)
  - Linux with notify-send not installed: same {:error, :enoent} guard as macOS fallback
