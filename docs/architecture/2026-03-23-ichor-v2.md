# ICHOR V2 Architecture

**Date**: 2026-03-23
**Status**: DRAFT
**Input**: Signal ownership audit, supervision tree trace, dead code analysis, SessionManager research, session findings

---

## Principles

1. **Sessions, not agents.** An agent is a blueprint. A session is a running process. Fleet owns sessions. Workshop owns blueprints.
2. **Emit is pure.** Signals broadcast facts. No reactions in the emit path. Subscribers react. Oban persists intent.
3. **One owner per concept.** If two modules think they own the same thing, one of them is wrong.
4. **Simple over clever.** Elixir Registry, DynamicSupervisor, tmux. No custom GenServer wrappers for things OTP already does.
5. **Ash for boundaries, plain Elixir for plumbing.** Ash Resources define the domain API. Internal mechanics are functions and processes.

## Domains

### Workshop (design-time, persisted)

What agents look like before they run. SQLite-backed CRUD.

| Resource | Table | Purpose |
|----------|-------|---------|
| AgentType | workshop_agent_types | Archetype: coordinator, lead, worker, researcher |
| Agent | workshop_agents | Named agent with type, model, instructions. belongs_to AgentType, optional belongs_to Team |
| Team | workshop_teams | Named group of agents with a strategy. has_many Agents |
| Prompt | workshop_prompts | Reusable prompt templates |

No runtime state. No GenServers. No signals. Pure data.

### Fleet (run-time, in-memory)

Running sessions. Elixir Registry + DynamicSupervisor + tmux.

| Concept | Mechanism | Purpose |
|---------|-----------|---------|
| Session | Registry entry + tmux pane | A live Claude/Gemini/Codex process |
| SessionSupervisor | DynamicSupervisor | Owns session processes |
| Registry | Elixir Registry | Lookup by session ID |

Session lifecycle: `spawn` creates a tmux session and registers it. `stop` kills the tmux session and unregisters. No GenServer per session unless the session needs mailbox or state -- tmux IS the process.

Note: Fleet.Session, Fleet.Supervisor, and Fleet.Preparations.LoadSessions were removed in commit f20ac4b. The active fleet implementation lives in Infrastructure (fleet_supervisor.ex, agent_process.ex, etc.).

```
Workshop.Team blueprint
  |
  v
Fleet.spawn_team("mes", team_blueprint)
  |
  v
For each agent in team:
  tmux new-session -> Registry.register -> done
```

### Signals (transport only)

The nervous system. Broadcasts facts. Does not react.

| Module | Purpose |
|--------|---------|
| Signals | Ash Domain + facade (emit/subscribe) |
| Signals.Runtime | PubSub broadcast + telemetry |
| Signals.Catalog | 143 signal definitions |
| Signals.Message | Envelope struct |
| Signals.Topics | Topic string builders |
| Signals.FromAsh | Ash notifier -> signal bridge |
| Signals.Buffer | Ring buffer for /signals UI page |

Everything else currently in `signals/` moves out:
- EventStream -> Events domain (or standalone)
- Bus -> Fleet (message delivery is session-scoped)
- AgentWatchdog -> Monitoring domain
- EntropyTracker -> Monitoring domain
- ProtocolTracker -> Monitoring domain (or delete -- barely functional)
- SchemaInterceptor -> deleted in commit f20ac4b

### Events (ingestion)

Hook events from Claude/Gemini/Codex arrive via HTTP. Normalized, buffered, emitted.

| Module | Purpose |
|--------|---------|
| Events.Stream | GenServer: ETS buffer, normalize, emit :new_event |
| Events.Normalizer | Pure: raw hook map -> canonical event |
| Events.Lifecycle | Pure: emit session_started/ended/team signals |

One concern per module. Stream owns the buffer. Normalizer transforms. Lifecycle emits.

### Monitoring (reactive observation)

Subscribers that watch the system and emit health signals. No direct actions -- observe and report.

| Module | Purpose |
|--------|---------|
| Monitoring.Watchdog | 5s tick: heartbeat, crash detection, escalation, pane scan |
| Monitoring.Entropy | Loop detection per session |
| Monitoring.Supervisor | Supervises watchdog + entropy |

Watchdog does NOT reassign tasks or write to Inbox. It emits `:agent_crashed`, and subscribers in the appropriate domain react.

## Signal Flow

```
HTTP hook event
  -> EventController
  -> Events.Stream.ingest_raw/1
  -> normalize, buffer in ETS
  -> Signals.emit(:new_event)
  -> subscribers react:
       Monitoring.Watchdog (health tracking)
       Monitoring.Entropy (loop detection)
       Archon.SignalManager (attention queue)
       Signals.Buffer (UI feed)
       MemoriesBridge (knowledge graph)
```

```
Ash resource mutation
  -> FromAsh notifier (post-commit)
  -> Signals.emit(:pipeline_created, etc)
  -> subscribers react
```

```
Watchdog detects crash
  -> Signals.emit(:agent_crashed)
  -> subscriber in Factory: reassign tasks (via Oban)
  -> subscriber in Archon: notify operator
  -- watchdog does NOT reassign or notify directly
```

## Message Delivery

Fleet owns message delivery. Messages go to sessions, not "agents."

```
Fleet.send_message(session_id, content)
  -> Registry.lookup(Ichor.Fleet.Registry, session_id)
  -> get tmux target from metadata
  -> Tmux.deliver(target, content)
  -> Signals.emit(:message_delivered)
```

No Bus GenServer. No ETS message log. Message delivery is a function call: look up the session, send to tmux, emit a signal.

## Migration Path

Not big-bang. Incremental:

1. **Workshop CRUD** (in progress) -- 4 persisted resources
2. **Fleet shell** -- Registry + DynamicSupervisor + Session resource with spawn/stop
3. **Events extraction** -- Move EventStream out of signals/ into events/
4. **Monitoring extraction** -- Move Watchdog + Entropy out of signals/
5. **Bus replacement** -- Fleet.send_message replaces Bus.send
6. **Old system removal** -- Trash Infrastructure.AgentProcess, FleetSupervisor, TeamSupervisor
7. **Dead code sweep** -- Catalog cleanup, delete never-emitted signals
8. **Dashboard rewire** -- Point dashboard_state.ex at Fleet + Workshop

Each step compiles clean. Each step has a rollback path. No step depends on more than one other step.
